1- Distribuição em Sistemas Computacionais


A - Fundamentos de Sistema Distribuídos


O sistema distribuido é o tipo de sistema computacional aonde o processamento é realizado em mais de um local, ao mesmo tempo (paralelamente) ou sequencialmente.

Para o usuário, independente de paralelo ou sequencial, será visto como um único sistema

O sistema distribuído, para poder funcionar, trabalha em camadas

O sistema distribuído é complexo, assim, é importante compreender este trabalho em camadas

Cada camada trata de um aspecto do sistema distribuído:


- A primeira camada (mais alta), é a camada de aplicativos e serviços

Essa camada trata, por exemplo, as interfaces dos programas, o que o usuário enxerga do sistema; serviços oferecidos pelo banco de dados (que é um aspecto da comunicação)

- A segunda camada é um middleware.

O middleware é um software que vai fazer a interface entre o aplicativo e a plataforma.

Para que o desenvolvolvimento seja feito de uma forma mais suave, sem se preocupar com um hardware especcífico, eu posso criar em uma máquina virtual (middleware)

o Middleware vai fazer exatamente esta transição, vai fazer a tradução daquilo que foi desenvolvido de uma forma mais genárica pra um determinado hardware

* a plataforma é o sistema operacional, computador e hardware de rede

- A terceira camada é o Sistema Operacional

- Na quarta camada (camada mais baixa do sistema) se tem o hardware propriamente dito

* essas camadas são feitas de modo a ocultar da camada superior as particularidades da camada inferior

* para permitir essa ilusão, os diversos componentes precisam ser ligados por uma rede, que a partir de seus protocolos padronizados irá mascarar as diferenças de suas arquiteturasf

B - Camadas de um Sistema Distribuídos


Um sistema distribuído pode, de forma simplificada, ser dividido em quatro camadas:


  1. Aplicativo ou serviço: o programa fornece o servço ou funcionalidade

  2. Middleware: software localizado entre o sistema operacional e os aplicativos cuja função é gerenciar os dados e comunicação entre as duas camadas, mascarando desta forma a heterogeinedidade da plataforma utilizada

  3. Sistema operacional: software de gerenciamento de hardware da máquina hospedeira

  4. Computador e hardware de redes : componentes eletrÇonicos que permitem o funcionamento do equippamento utilizado

C - Tipos de Sistemas Computacionais


Se originaram em 1950 e foram evoluindo ao longo dos anos, inicialmente sendo centralizados e entçai evoluindo para paralelos

Sistema centralizado possui um grande computador central, o mainframe, ao qual o usupario se ligava utilizando um terminal, uma linha de comunicação

por muitos era conhecido como terminal brro pois este terminal não possuia capacidade de processamento, era o monitor, teclado e controlador de comunicação - minimodem. o usuário digitava, apertava uma tecla, o código da tecla ia para o computador central que interpretava e mandava de colta o que era pra ser exibido localmente, nada era processado localmene

com o advento das redes de computadores, surgiram os sistemas cliente-servidores, onde há a figura de um servidor central, com varias máquinas interligadas em uma rede de computadores, todos interligados aos servidores, trocando informações

em paralelo, acontecia a evoolução do hardware dos processadores, surgindo assim o ssistmea paralelo

o sistema paralelo é aquele em que possuo mais de um processador executando as atividades ao mesmo tempo, ou seja executando mais de um processamento paralelamente

inicialmente era algo caro em que se colocava vários processadores em uma máquina e eles colaboravam um com o outro, sendo um sistem paralelo fortemente acoplado, pois a interligação entre os processadores se dava por meio de um barramento interno - o qual é altamente veloz

D - Sistemas Centralizados


Este é o tipo mais antigo, no qual há a figura de um grande computador central, o mainframe, que recebe e processa uma grande quantidade de informações

A mecâmica de seu funcionamento consiste em o usuário acessar o mainframe e realizar a sua solicitação. A servidor, por sua vez, processa a requisição, acessa os recursos necessários e produz o resultado


Sistemas Paralelos


Neste tipo de sistema, existem vários processamentos ocorrendo ao mesmo tempo para produzir os resultados solicitados


Este processamento simultâneo das várias partes de uma aplicação pode ser realizado das sequintes formas:


  • Em uma máquina com vários processadores

  • Em várias máquinas interligadas por uma rede de comunicação de dados se comportando como um único computador

  • Em uma máquina com um único processador com vários núcleos, como nas CPUs mais modernas


Em sistemas paralelos, uma aplicação que implica o upload de um arquivo pelo usuário pode ser dividida em duas partes processadas ao mesmo tempo, uma que faz a carga da imagem e outra que mostraá a barra de progresso do carregamento


Em comparação com sistemas centralizados, a arquitetura paralela é de gerenciamento mais difícil e necessita de implementação mais robusta em sua segurança


Os sistemas paralelos podem ser de dois tipos:



Sistemas paralelos fortemente acoplados

Um grande conjunto de sistemas computacionais corresponde a este tipo - qualquer tipo de computador equipado com mais de um processador ou, se tiver um único processador, que este possua em sua arquitetura mais de um núcleo

Um sistema paralelo fortemente acoplado corresponde, praticamente a todos os atuais equipamentos que utilizamos para realizar a execução de aplicativos

Em sistemas fortemente acoplado, a comunicação entre os processadores ocorre utilizando o barramento interno do equipamento e as várias CPUs/ núcleos compartilham a memória principal (conhecida popularmente como RAM) do equipamento

Como a arquitetura suporta vários processadores, ou núcleos, vários programas, ou partes diferentes de um mesmo programa, podem ser executados de forma simultânea


Sistemas Paralelos Fracamente Acoplados

Este tipo se aplica aos sistemas de maior escala, sempre conectados por rede de computadores. Ele é composto de máquinas distintas, podendo cada uma delas ser individualmente do tipo fortemente acoplada, que trabalham em conjunto, de forma coordenada, para realizar o processamento solicitado e produzir o resultado pretendido

Exemplos típicos de sistemas paralelos fracamente acoplados:
Arquitetura cliente servidor
Essa é a arquitetura de rede mais conhecida e utilizada nos sistemas informatizados. Nela está presente a figura do computador que fornece os serviços, denominado servidor, e do computador que demanda o uso do serviço, denominado cliente
Arquitetura Ponto a Ponto
Também conhecida como arquitetura peer-to-peer, ou simplesmente, p2p, tem como característica o fato de não existir a figura do servidor e do cliente de forma marcada. Todas as máquinas da rede podem eventualmente fornecer um serviço ou demandar o serviço oferecido por outra máquina
Em relação aos sistemas fortemente acoplados, os fracamente acoplados possuem algumas vantagens e desvantagens
  • Vantagem: A flexibilidade e a escalabilidade, sendo muito mais simples acrescentar mais poder computacional, pois basta realizar o acréscimo de novas máquinas
  • Desvantagem: A velocidade de transferência de dados entre os processadores é muito menor, já que nenhuma rede consegue atingir a velocidade de transferência dos barramentos internos de uma máquina
Os sistemas distribuídos são enquadrados neste tipo, já que são compostos por várias máquinas, interligadas por uma rede de computadores, que trabalham em conjunto processando as requisições
