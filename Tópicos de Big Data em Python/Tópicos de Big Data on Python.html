Fundamentos da Tecnologia de Big Data


O termo Big Data é usado em referencia à aplicações que envolvem grandes volumes de dados. Big Data é um conjunto de tecnologias que gerenciam aplicações, as quais, além do grande volume de dados, trabalham com dados que podem ser gerados com alta velocidade, de diversas fontes e diferentes formatos.


Big Data é o tratamento especializado de armazenamento, gerenciamento e análise demandados por crescentes volumes de dados e toda a complexidade que envolve estes dados

Os dados precisam ser tratados por um ciclo de vida para que seja possivel a extrção de informações úteis, transformando tais informações em conhecimento

---

Devido ao potencial de valor que as aplicações de Big Data podem gerar, tanto empresas quanto agÇencias governamentais têm investido nessa área por meio do desenvolvimento de soluções que capturem dados com mais qualidade para facilitar as etapas posteriores de armazenamento, gerenciamento e análise

---

Dados que surgem de variadas fontes proporcionam um mapeamento mais detalhado do comportamento das pessoas, o que deve ser amparado pelas searas ético-legais.

No Brasil, um dos mecanismos de amparo legal se dá por meio do disposto constitucional de inviolabilidade do sigilo de dados e comunicações, bem como pela Lei Geral de Proteção de Dados - Lei n 13709/2018

Arquitetura básica de Big Data

uma das diferenças entre o Big Data e os bancos de dados tradicionais é a dificuldade do ultimo em lidar com operações de dados em sistemas heterogêneos Estes sistemas heterogÇeneos são os Data Lakes, que são um enorme repositório de arquivos e objetos de dados --- As soluções de Big Data precisam ser eficientes para que possam produzir resultados ábeis em um tempo de resposta aceitável --- Componentes da Arquitetura Big Data 1- Fontes de Dados - Data Sources Os sistemas de Big Data podem ser alimentaos tanto por dados tradicionais quanto por meio de dados que estão na nuvem e produzidos por sistemas de Internet das Coisas - IoT. Em muitos casos, esse processo, chamado de aquisição de dados, ocorre em tempo real. 2- Armazenamento de dados - Data Storage Os dados precisam ser aramzenados de forma eficiente para otimizar seu acesso e sua segurnaça esse armazenamento pode ser feito de diversas formas - na nuvem, em banos de dados estruturados ou em bancos de dados não estruturados que tenham: - escalabilidade: capacidade de crescer com consistencia - disponibilidade: prontos para acesso sempre que forem demandados - segurança: mecanismos que garantam privacidade e restrição de acesso - padronização: armazenamento seguindo um padrão que facilite sua repercução posteriormente 3- Processamento em lote (batch processing) é o armazenamento dos dados em lotes para que, então, seja realizado o processamento. Isto é feito deste modo para que se lide com grandes volumes de dados quando não é viável fazer o processamento dos dados em fluxo 4- Ingestão de mensagens (message ingestion) é o agrupamento de dados de modo a trazer tal agrupamento para um sistema de processamento, processamento este no qual os dados deste agrupamento podem ser armazenados, analisados e acessados 5- processamento de fluxo (stream processing) é o processamento dos dados sendo feitos à medida em que os dados são produzidos ou recebidos. Essa situação ocorre com frequência em processos em eventos produzidos por sensores, atividades do usuário em um site e negociações financeiras, que tÇem em comum o fato dos dados serem criados como uma série de eventos de fluxo contínuo 6- armazenamento de dados analíticos (analytical data store) é o processo de armazenamento de dados que negócios, mercado e clientes para posterior análise. Aqui as aplicações desses dados são chamadas de Business Intelligence - BI. Aqui os bancos de dados analíticos são otimizados para que as consultas sejam rápidas e eficiÇentes 7- Análise e relatórios (analysis and reporting) os relatórios são uma organização dos dados com o objetivo de fazer resumos informativos e monitorar o desempenho de diferentes áreas da empresa. a análise consiste em explorar dados e relatórios para extrair informações que agreguem valor e que possam ser usadas para compreender melhor os negócios e aperfeiçoar seu desempenho Os 5 Vs do Big Data 1 - Volume: esta característica está relacionada com a escala da geração e coleta de massas de dados. A percepção de grandes volumes de daods está relacionada com a tecnologia disponível em determinado momento.
